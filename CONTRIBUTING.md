# Contributing to the Ohlone Unicode Project

Thank you for your interest in supporting digital infrastructure for Ohlone languages.

## Guiding Principles

1. **Tribal sovereignty over language**: Ohlone tribal communities have final authority over their languages and orthographic decisions. This project supports — never replaces — community-led revitalization.

2. **Published sources only**: All character inventories and orthographic descriptions are based on published academic works or publicly available tribal resources. We do not include unpublished or restricted language data.

3. **Accuracy over speed**: Encoding decisions affect language preservation for decades. We prioritize getting it right.

## How to Contribute

### If you are an Ohlone tribal member or language learner

Your knowledge is the most valuable contribution. You can help by:

- Reviewing character inventories for accuracy
- Identifying characters or sounds missing from our documentation
- Testing keyboard layouts and providing feedback
- Sharing publicly available learning resources

### If you are a linguist

- Verify phonemic inventories against primary sources
- Review IPA transcriptions
- Help draft CLDR locale data
- Identify additional orthographic conventions from published sources

### If you are a software engineer

- Build keyboard layouts (macOS Ukelele, Windows MSKLC, Linux XKB)
- Improve the validation and transliteration tools
- Create input methods for mobile platforms
- Test font coverage across platforms

### If you are a font designer

- Ensure glyph coverage for all required Ohlone characters
- Design glyphs that distinguish U+02BC from U+0027 visually
- Consider creating an open-source Ohlone-optimized font

## Submitting Changes

1. Fork the repository
2. Create a descriptive branch name (e.g., `fix-rumsen-char-table`)
3. Make your changes with clear commit messages
4. Open a pull request describing what you changed and why
5. Reference any sources for factual changes

## Code of Conduct

- Be respectful of Indigenous languages and communities
- Do not share restricted or unpublished language data
- Credit tribal communities and linguists whose work you reference
- Welcome feedback from community members graciously

## Questions?

Open an issue on GitHub or reach out to the project maintainers.
